05 Technical Debt & Known Issues

🔴
No automated test coverage
Neither frontend nor backend have test suites. Any change to the system carries unquantified regression risk. Addressed in R3.
🔴
No persistent storage
JSON file-based data layer means no writes survive a server restart. Not suitable for production use.
🟡
Mixed Vue API styles
Reports.vue used Options API while the rest of the codebase uses Composition API. Remediated in R1.
🟡
No authentication
The application has no login or access control. Any user on the network can access all data and operations.
🟡
CORS allows all origins
Backend is configured with allow_origins=["*"]. Must be restricted to specific origins before any internet-facing deployment.
🟡
Incomplete endpoints
Tasks API is partially implemented on the client but not in the backend. Purchase Orders backend was missing; added in R2.
🟡
Partial i18n coverage
LanguageSwitcher and locale files exist (en/ja) but i18n is not applied to all modules. Addressed in D2.
🟢
No pagination
All API responses return full datasets. Acceptable at current data volume (~250 orders, 32 inventory items); will need addressing as data grows.
